# 저니픽(Product Requirements Document)

## 1. 제품 개요
저니픽은 한국을 방문한 외국인 관광객이 로컬 Planner가 기획·운영하는 문화 체험 상품을 쉽고 빠르게 탐색하고, 원하는 일정에 맞춰 예약·결제할 수 있는 웹 기반 마켓플레이스입니다. Planner는 간단한 등록·승인 절차를 통해 전 세계 고객에게 체험을 판매할 수 있습니다.

## 2. 목표
1. 외국인 관광객이 로컬 체험 상품을 한 곳에서 비교·예약할 수 있는 원스톱 플랫폼 제공
2. Planner가 손쉽게 상품을 등록·관리하고 매출을 창출하도록 지원
3. 관리자(최고/중간)가 판매 현황·품질·승인 절차를 효과적으로 관리

## 3. 핵심 사용자 및 페르소나
| 역할 | 주요 니즈 | 설명 |
|---|---|---|
| Tourist(주사용자) | 믿을 수 있는 로컬 체험을 발견하고 즉시 예약·결제하고 싶다 | 20~40대 해외 여행자, 모바일·웹 친숙, 카드 결제 선호 |
| Planner(판매자) | 해외 고객에게 체험을 노출하고 간편하게 판매·스케줄 관리 | 프리랜서/소규모 업체, 영어 가능, 모바일 우선 |
| 중간관리자 | Planner가 올린 상품을 신속히 검수·승인/반려 | 품질 담당 직원 |
| 최고관리자 | 매출·수수료·판매 데이터를 모니터링해 비즈니스 의사결정 | 운영 총괄 |

## 4. 핵심 기능 범위 (MVP)
1. 상품 탐색·상세·결제
   - 리스트: 검색, 가격/카테고리/지역/키워드 필터 사이드바, 카드 및 리스트 뷰
   - 상세: 이미지(썸네일+슬라이드), 가격, Overview(WYSIWYG), Location(구글맵+설명), Additional Info, 선택 달력, 1달러 티켓 옵션, 리뷰
   - 결제: Stripe PG 연동, 옵션·수량 선택, 영수증 이메일 발송
2. 다단계 역할 관리 & 승인 워크플로우
   - Planner 상품 등록 폼(WYSIWYG, 이미지 업로드, 달력 일정 지정)
   - 중간관리자 승인/반려 알림(이메일+내부 알림) 및 상태 관리
3. 마이페이지 & 개인화
   - 찜한 상품, 예약 내역, 리뷰 작성·관리, 프로필 수정
4. 통계·매출 대시보드(최고관리자)
   - 기간별 매출, 판매 Top Planner, 수수료, 상품별 판매수량
5. 리뷰 & 평점 시스템(추가 기능)
   - 예약 완료 고객만 작성 가능, 별점+코멘트, Planner 답글 지원

## 5. 주요 유저 스토리 (발췌)
- Tourist로서, 내가 원하는 날짜에 가능한 한식 쿠킹 클래스를 검색해 바로 결제하고 싶다.
- Planner로서, 새로운 체험 상품을 등록하고 관리자의 승인을 받아 판매를 시작하고 싶다.
- 중간관리자로서, 신규 등록 상품을 검수 후 승인 또는 반려하고 싶다.
- 최고관리자로서, 이번 달 총 매출과 Planner별 판매 실적을 대시보드에서 확인하고 싶다.

## 6. 성공 지표 (KPIs)
1. 월간 활성 구매자(MAB)  
2. 상품 상세 → 결제 전환율  
3. Planner 승인 후 첫 판매까지 평균 소요 시간  
4. 평균 리뷰 평점 / 리뷰 작성율

## 7. 개발 전략
- **MVP 우선 출시**: 위 핵심 기능(1~4) + 리뷰 시스템만 1단계로 구현 후, 사용자 피드백에 따라 다국어 지원·실시간 채팅 등 확장
- **Agile 2주 스프린트**: 백로그를 우선순위화하고 지속적인 사용자 테스트 적용

## 8. 범위 외(차기 버전 고려)
- 다국어 자동 번역 지원
- 실시간 채팅/문의
- 프로모션 쿠폰/할인 코드 관리

## 9. 기술 스택(선정 배경)
- 프론트엔드: Next.js + TypeScript (SSR & SEO, 학습 난이도 낮음)
- 백엔드/DB: Supabase(PostgreSQL) (Auth, Storage, Realtime 기능 포함)
- 결제: Stripe
- 배포: Vercel (Next.js 친화적)

## 10. 리스크 & 가정
- 해외 결제 시 PG 규정 변경 가능성 → Stripe 최신 가이드 모니터링
- Planner 콘텐츠 품질 편차 → 리뷰·평점, 승인 절차로 보완

## 11. 일정(예시)
| Phase | 기간 | 산출물 |
|---|---|---|
| 기획 확정 | 주 1 | PRD, 와이어프레임 |
| MVP 개발 | 주 2~8 | 기능 구현 & 테스트 |
| 베타 런칭 | 주 9 | 초기 사용자 온보딩 |
| 피드백 개선 | 주 10~ | 반복 개선 |
